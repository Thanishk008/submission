SECTIONS 1–3: INTRODUCTION & PROBLEM STATEMENT
──────────────────────────────────────────────────────────────────────────────

1. Problem: Credit card fraud detection as binary classification
   - Highly imbalanced: only 0.172% positive (fraud) class
   - Real-world impact: financial losses, customer trust
   - Challenge: achieving high recall (catch fraud) without
     flooding investigators with false positives (precision)

2. Approach summary:
   - Baseline: Logistic Regression with class weighting
   - Primary model: 1D CNN with SMOTE oversampling
   - Ablation: effect of SMOTE, Dropout rate, BatchNorm
   - Foundation model: MobileNetV2 (ImageNet pre-trained)
   - Evaluation: Precision, Recall, F1, ROC-AUC, PR-AUC

3. Why 1D CNN for tabular data?
   - Conv1D treats 30 features as a 1D sequence
   - Captures local feature interactions via sliding kernel
   - Lighter than 2D approaches, avoids artificial reshaping
   - BatchNorm + Dropout provide regularization for small datasets

──────────────────────────────────────────────────────────────────────────────
SECTION 4: DATA PROCESSING
──────────────────────────────────────────────────────────────────────────────

1. Dataset: Kaggle Credit Card Fraud Detection (mlg-ulb/creditcardfraud)
   - 284,807 transactions, 30 features (V1-V28 PCA + Time + Amount)
   - 492 fraud (0.172%), 284,315 legitimate

2. Feature scaling: StandardScaler (zero mean, unit variance)
   - Applied on all 30 features including Time and Amount
   - V1-V28 are already PCA-transformed but not standardized

3. Train/test split: 70/30 stratified
   - 199,364 train / 85,443 test
   - train_test_split(stratify=y) preserves the 0.17% fraud ratio

4. SMOTE applied AFTER split (prevents data leakage)
   - Applied only to training set; test set stays unaugmented
   - k_neighbors=5, after SMOTE: ~318K training samples

5. Known limitation: StandardScaler fit on full dataset before split
   - scaler.fit_transform(X) includes test data statistics
   - For PCA features this has negligible effect, but mention as
     a limitation. Ideally fit only on training set.

──────────────────────────────────────────────────────────────────────────────
SECTION 5.1: BASELINE MODEL
──────────────────────────────────────────────────────────────────────────────

6. Logistic Regression with class_weight='balanced'
   - Solver: 'lbfgs', max_iter=1000
   - Trained on raw (non-SMOTE) data — class_weight handles imbalance
   - sklearn adjusts weights inversely proportional to class frequency

──────────────────────────────────────────────────────────────────────────────
SECTION 5.2: CNN MODEL ARCHITECTURE
──────────────────────────────────────────────────────────────────────────────

7. 1D CNN: 3 conv blocks + 2 dense layers
   - Conv Block: Conv1D → BatchNorm → MaxPool1D(2) → Dropout
   - Filters: 32 → 64 → 128 (doubling), kernel_size=3, padding='same'
   - Dense head: Flatten → Dense(256) → Dense(128) → Dense(1, sigmoid)
   - All activations ReLU except final sigmoid

8. Dense dropout is higher than conv dropout
   - Dense dropout = min(dropout_rate + 0.1, 0.5)
   - E.g., conv dropout=0.3 → dense dropout=0.4
   - Rationale: Dense layers have more parameters, need stronger
     regularization

9. Ablation design: BatchNorm/Dropout layers are conditionally absent
   - When disabled, layers are removed entirely (not zeroed)
   - Different ablation variants have different param counts

──────────────────────────────────────────────────────────────────────────────
SECTION 5.3: TRAINING DETAILS & REPRODUCIBILITY
──────────────────────────────────────────────────────────────────────────────

10. Optimizer: Adam, lr=0.001, binary cross-entropy loss
    - Consistent across all models

11. Callbacks:
    - EarlyStopping: patience=5, restore_best_weights=True on val_loss
    - ReduceLROnPlateau: factor=0.5, patience=3, min_lr=1e-6

12. Epochs: train.py uses 50, ablation uses 30 (uniform for fair comparison)

13. Threshold = 0.5 for classification (not tuned)

14. Hardware: CPU-only, TF 2.20.0, Python 3.9.5, Windows
    - RTX 3060 mobile available but TF 2.11+ dropped native Windows
      GPU support

15. Seed=42 everywhere (np, tf, sklearn) for reproducibility

──────────────────────────────────────────────────────────────────────────────
SECTION 6: PERFORMANCE IMPROVEMENT TECHNIQUES (ABLATION)
──────────────────────────────────────────────────────────────────────────────

16. Technique 1 — SMOTE: "Without SMOTE" has best F1 (0.7930)
    - Counterintuitive: SMOTE's synthetic samples introduce noise
    - Without SMOTE: precision jumps 0.2993 → 0.8248 (fewer FPs)
    - Trade-off: recall drops 0.8311 → 0.7635 (misses more fraud)
    - Insight: real examples may be more informative than synthetic ones

17. Technique 2 — Dropout: 0.0 achieves second best F1 (0.7557)
    - SMOTE-augmented dataset provides enough implicit regularization
    - Dropout=0.5: F1 drops to 0.2349 — too aggressive, underfits

18. Technique 3 — BatchNorm: critical for this model
    - Without BatchNorm: F1 drops to 0.2153
    - Training is unstable without it — performance degrades heavily

SECTION 8: COMPARISON OF APPROACHES
──────────────────────────────────────────────────────────────────────────────

19. LR has higher ROC-AUC (0.9683) than CNN (0.9543) — expected
    - ROC-AUC measures ranking ability across ALL thresholds
    - LR is well-calibrated but at the 0.5 threshold its precision
      collapses (0.0673) → low F1 (0.1249)
    - CNN has better performance at the operating threshold

──────────────────────────────────────────────────────────────────────────────
SECTION 9: FOUNDATION MODEL COMPARISON
──────────────────────────────────────────────────────────────────────────────

20. MobileNetV2 (alpha=0.35, ~0.4M params) as foundation model
    - Pre-trained on ImageNet, frozen backbone + trainable Dense head:
      Dense(128, relu) → Dropout(0.4) → Dense(1, sigmoid)

21. Tabular → 2D image reshaping
    - 30 features → 6×5×1 image → tiled to 3 channels → resized to 32×32
    - Required because MobileNetV2 expects min 32×32 RGB input

22. class_weight='balanced' instead of SMOTE (foundation model only)
    - SMOTE would inflate dataset; 2D convolutions are expensive per sample
    - class_weight achieves same imbalance handling without inflation

23. Why 1D CNN beats MobileNetV2 (KEY for analysis, worth 3 pts):
    - Domain mismatch: ImageNet features (edges, textures, shapes) are
      irrelevant to tabular fraud data reshaped into synthetic images
    - Frozen backbone can't adapt its filters to tabular data
    - Information loss from resizing 6×5 → 32×32 (interpolation artifacts)
    - 1D CNN is purpose-built for sequential tabular features
    - Quote: "Transfer learning is most effective when the source and
      target domains share similar data characteristics."

SECTION 10: ERROR & FAILURE ANALYSIS
──────────────────────────────────────────────────────────────────────────────

24. Most errors are high-confidence
    - FNs: P(Fraud) ≈ 0.0 — model confidently labels fraud as legit
    - Top FP: P(Fraud) = 1.0 — model confidently labels legit as fraud
    - The model isn't uncertain — it's systematically fooled by edge cases

25. FN samples cluster in "normal" feature ranges
    - On V14, V17, V12, V10, V4: missed fraud looks like legitimate data
    - These fraud transactions lack the typical fraud signature
