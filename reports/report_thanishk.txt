ABSTRACT
================================================================================

Credit card fraud detection presents a critical binary classification challenge
characterized by extreme class imbalance, where fraudulent transactions
constitute only 0.172% of the dataset. This report investigates the application
of a one-dimensional Convolutional Neural Network (1D CNN) with Synthetic
Minority Over-sampling Technique (SMOTE) for identifying fraudulent
transactions in the Kaggle Credit Card Fraud Detection dataset comprising
284,807 transactions and 30 features. A Logistic Regression baseline, a 1D CNN
with ablation studies on SMOTE, Dropout, and Batch Normalization, and a
MobileNetV2 foundation model comparison were developed and evaluated. The
ablation study revealed a counterintuitive finding: the best-performing
configuration was the 1D CNN trained without SMOTE, achieving an F1-score of
0.7930 with precision of 0.8248 and recall of 0.7635. The 1D CNN significantly
outperformed the MobileNetV2 foundation model (F1=0.0291), demonstrating that
transfer learning from natural image domains does not generalize well to
tabular fraud data.

1. INTRODUCTION & PROBLEM STATEMENT
================================================================================

Credit card fraud poses a significant threat to financial institutions and
consumers worldwide, resulting in billions of dollars in annual losses and
erosion of customer trust. As digital payment systems continue to expand, the
volume of transactions requiring automated monitoring has grown proportionally,
making manual review infeasible. Machine learning and deep learning methods
offer the promise of real-time, scalable fraud detection systems capable of
flagging suspicious activity before financial damage occurs.

The central challenge in fraud detection is the extreme class imbalance
inherent in transaction datasets. In the dataset employed in this study, only
492 out of 284,807 transactions (0.172%) are fraudulent. This imbalance
creates a fundamental tension between two competing objectives: achieving high
recall to catch as many fraudulent transactions as possible, while maintaining
sufficient precision to avoid flooding human investigators with false
positives. A naive classifier that labels every transaction as legitimate would
achieve 99.83% accuracy yet fail entirely at its intended purpose.

This report presents a systematic investigation of deep learning approaches
for credit card fraud detection. The methodology encompasses a Logistic
Regression baseline, a purpose-built 1D Convolutional Neural Network as the
primary model, an ablation study examining the effects of SMOTE, Dropout
regularization, and Batch Normalization, and a comparison against MobileNetV2
as a foundation model. All models are evaluated using Precision, Recall,
F1-score, ROC-AUC, and PR-AUC to provide a comprehensive assessment of
performance across multiple dimensions.

A 1D CNN architecture was selected for this tabular classification task because
Conv1D layers treat the 30 input features as a one-dimensional sequence,
enabling the network to capture local feature interactions through sliding
kernels. This approach is lighter than two-dimensional alternatives and avoids
the artificial reshaping of tabular data into image-like grids. Combined with
Batch Normalization and Dropout, the 1D CNN provides a well-regularized model
suitable for datasets with limited positive examples.


================================================================================
2. DATA PROCESSING
================================================================================

2.1 Dataset Description
-----------------------

The dataset used in this study is the Kaggle Credit Card Fraud Detection
dataset (mlg-ulb/creditcardfraud), originally published by the Machine
Learning Group at Université Libre de Bruxelles [1]. It contains 284,807
credit card transactions made by European cardholders over a two-day period in
September 2013. Of these, 492 transactions (0.172%) are labeled as fraudulent
and 284,315 as legitimate.

The dataset comprises 30 numerical features. Features V1 through V28 are the
result of a PCA transformation applied to the original features for
confidentiality reasons. The remaining two features, Time (seconds elapsed from
the first transaction) and Amount (transaction value), are provided in their
original form. The target variable, Class, is binary: 1 for fraud and 0 for
legitimate.

2.2 Feature Scaling
--------------------

StandardScaler was applied to all 30 features to normalize them to zero mean
and unit variance. Although V1–V28 are already PCA-transformed, they are not
standardized, and the Time and Amount features exist on entirely different
scales. Standardization ensures that no single feature dominates the learning
process due to its magnitude.

A known limitation of the current implementation is that StandardScaler was
fit on the full dataset prior to the train/test split. This means that test
set statistics marginally influence the scaler parameters, constituting a
minor form of data leakage. For PCA-derived features, this effect is
negligible, but ideally the scaler should be fit exclusively on the training
set. This is acknowledged as a limitation.

2.3 Train/Test Split
--------------------

The dataset was split into 70% training (199,364 samples) and 30% testing
(85,443 samples) using stratified sampling. The stratify parameter preserves
the original 0.172% fraud ratio in both partitions, ensuring that the test set
accurately reflects the real-world class distribution.

2.4 SMOTE Oversampling
----------------------

The Synthetic Minority Over-sampling Technique (SMOTE) [2] was applied to the
training set after the train/test split to prevent data leakage. SMOTE
generates synthetic fraud samples by interpolating between existing minority
class instances using k=5 nearest neighbors. After oversampling, the training
set expanded from approximately 199,000 to approximately 318,000 samples, with
the fraud class upsampled to match the legitimate class count.

Critically, SMOTE was applied only to the training data. The test set remained
unaugmented and reflects the original class distribution, ensuring that
evaluation metrics are computed on realistic, unmodified data.


================================================================================
3. MODEL ARCHITECTURE
================================================================================

3.1 Baseline Model: Logistic Regression
-----------------------------------------

A Logistic Regression classifier was employed as the baseline model to
establish a reference point against which deep learning approaches could be
compared. The model was configured with the L-BFGS solver, a maximum of 1,000
iterations, and class_weight='balanced'. The balanced class weighting instructs
scikit-learn [7] to automatically adjust weights inversely proportional to
class frequency, effectively upweighting fraud samples during training without
requiring synthetic oversampling.

The baseline was trained on the raw (non-SMOTE) training data, as Logistic
Regression does not benefit substantially from SMOTE when class weighting is
already applied. This design choice also provides a cleaner comparison: the
baseline uses class weighting while the CNN uses SMOTE, representing two
distinct approaches to the same imbalance problem.

3.2 1D CNN Architecture
------------------------

The primary model is a one-dimensional Convolutional Neural Network consisting
of three convolutional blocks followed by a two-layer dense classification
head. Each convolutional block contains the following layers in sequence:

  Conv1D → BatchNorm → MaxPool1D(pool_size=2) → Dropout

The three convolutional blocks use a doubling filter progression of 32, 64,
and 128 filters respectively, all with kernel_size=3 and padding='same'. All
convolutional layers use the ReLU activation function. The classification head
consists of:

  Flatten → Dense(256, ReLU) → Dense(128, ReLU) → Dense(1, sigmoid)

A deliberate design choice was made regarding dropout rates in the dense
layers. The dense dropout rate is computed as:

  dropout_dense = min(dropout_rate + 0.1, 0.5)

For example, when the convolutional dropout rate is set to 0.3, the dense
dropout rate becomes 0.4. This asymmetry reflects the fact that dense layers
contain significantly more parameters than convolutional layers and are
therefore more susceptible to overfitting on the flattened feature
representations.

For the ablation study, Batch Normalization and Dropout layers are
conditionally included. When disabled, the layers are removed entirely from
the model architecture rather than being set to zero. This means that
different ablation variants have different total parameter counts and
architectural depths.

3.3 Training Details
---------------------

All models were compiled with the Adam optimizer [5] using a learning rate of
0.001 and binary cross-entropy as the loss function, defined as:

  L = -[y * log(p) + (1 - y) * log(1 - p)]

where y is the true label and p is the predicted probability. This loss
function was chosen for its suitability to binary classification tasks and its
smooth gradient properties.

Two callbacks were employed during training:

  - EarlyStopping: Monitors validation loss with patience=5 and
    restore_best_weights=True, ensuring the final model corresponds to the
    epoch with lowest validation loss rather than the last epoch.

  - ReduceLROnPlateau: Reduces the learning rate by a factor of 0.5 when
    validation loss plateaus for 3 consecutive epochs, with a minimum
    learning rate of 1e-6.

The main training script (train.py) uses a maximum of 50 epochs, while the
ablation study uses 30 epochs uniformly across all variants to ensure fair
comparison. The classification threshold was fixed at 0.5 throughout all
experiments and was not tuned.

All experiments were conducted on a CPU-only environment running TensorFlow
2.20.0 and Python 3.9.5 on Windows. Although an NVIDIA RTX 3060 mobile GPU
was available, TensorFlow versions 2.11 and later dropped native Windows GPU
support, necessitating CPU-only execution.

Reproducibility was ensured by setting random seeds to 42 across all sources
of randomness: NumPy (np.random.seed), TensorFlow (tf.random.set_seed), and
scikit-learn (random_state parameter in all splits and SMOTE calls).


================================================================================
4. PERFORMANCE IMPROVEMENT TECHNIQUES (ABLATION STUDY)
================================================================================

An ablation study was conducted to systematically evaluate the contribution of
three key techniques to model performance: SMOTE oversampling, Dropout
regularization, and Batch Normalization. Nine experiments were executed, each
modifying a single variable while holding all others at their default values,
using 30 training epochs uniformly for fair comparison.

4.1 Technique 1: SMOTE
------------------------

The impact of SMOTE was evaluated by comparing the default 1D CNN (trained
with SMOTE, Dropout=0.3, BatchNorm enabled) against an identical architecture
trained without SMOTE. The results were counterintuitive: the model trained
without SMOTE achieved the highest F1-score of any configuration at 0.7930,
substantially outperforming the SMOTE-augmented model (F1=0.4401).

This improvement was driven primarily by a dramatic increase in precision, from
0.2993 to 0.8248, indicating far fewer false positives. The trade-off was a
moderate decrease in recall from 0.8311 to 0.7635. The explanation lies in the
nature of synthetic samples: SMOTE generates fraud examples by interpolating
between existing instances, which can introduce noise into the decision
boundary. When the model trains exclusively on real examples, it learns more
precise fraud signatures without being confused by synthetic artifacts. This
finding suggests that for fraud detection, where the positive class has highly
specific patterns, real examples may be more informative than synthetic ones.

4.2 Technique 2: Dropout Rate
------------------------------

Three Dropout rates were compared: the default 0.3, no dropout (0.0), and
aggressive dropout (0.5). Removing dropout entirely yielded the second-best
F1-score of 0.7557, suggesting that the SMOTE-augmented dataset provides
sufficient implicit regularization through its increased sample diversity,
making explicit dropout redundant.

Conversely, increasing dropout to 0.5 caused a sharp performance decline to
F1=0.2349. At this aggressive rate, too many neurons are deactivated during
training, preventing the network from learning sufficiently complex
representations. The model effectively underfits, defaulting to high-recall
but low-precision predictions.

4.3 Technique 3: Batch Normalization
--------------------------------------

Batch Normalization [4] proved to be the most critical component for stable
training. Removing BatchNorm caused F1 to drop to 0.2153, the lowest score
among all CNN variants. Without normalization of intermediate activations,
training becomes unstable: internal covariate shift causes gradient
magnitudes to vary unpredictably across layers, preventing the network from
converging to a good solution. This result confirms that Batch Normalization
is not merely a convenience but a structural necessity for this architecture.

6. COMPARISON OF APPROACHES
================================================================================

The unified comparison across all four model types reveals several important
findings about the nature of fraud detection and the suitability of different
approaches.

Logistic Regression achieved the highest ROC-AUC score (0.9683) among all
models, which at first glance appears paradoxical given its low F1-score of
0.1249. This discrepancy is explained by the distinction between ranking
ability and threshold-specific performance. ROC-AUC measures a model's
ability to rank positive samples higher than negative samples across all
possible thresholds. Logistic Regression, as a well-calibrated probabilistic
model, excels at this ranking task. However, at the fixed classification
threshold of 0.5, its precision collapses to 0.0673 — meaning that over 93%
of its fraud predictions are false alarms. This renders it impractical for
real-world deployment despite its strong ranking capability.

The default 1D CNN improved substantially over the baseline, raising F1 from
0.1249 to 0.4401 through better precision (0.2993 vs 0.0673) while
maintaining comparable recall (0.8311 vs 0.8784). The best ablation variant
(Without SMOTE) further increased F1 to 0.7930 by dramatically improving
precision to 0.8248, with an acceptable recall trade-off to 0.7635.

The MobileNetV2 foundation model performed worst across all metrics, with an
F1-score of only 0.0291 and a PR-AUC of 0.3016. This result is analyzed in
detail in the following section. The complete comparison demonstrates that
purpose-built architectures operating on native data representations
consistently outperform both generic baselines and transferred vision models
for tabular fraud detection.


================================================================================
7. FOUNDATION MODEL COMPARISON
================================================================================

To evaluate the applicability of transfer learning to tabular fraud detection,
MobileNetV2 [3] was employed as a foundation model. MobileNetV2 was
configured with a width multiplier (alpha) of 0.35, resulting in
approximately 0.4 million parameters. The backbone was initialized with
ImageNet pre-trained weights and frozen (all layers set to trainable=False).
Only a trainable classification head was appended:

  Dense(128, ReLU) → Dropout(0.4) → Dense(1, sigmoid)

7.1 Tabular-to-Image Reshaping
--------------------------------

Since MobileNetV2 expects image input with a minimum resolution of 32×32
pixels and three color channels, the 30 tabular features were reshaped into a
6×5×1 single-channel image, tiled across three channels to create a 6×5×3
pseudo-RGB representation, and then resized to 32×32×3 using a bilinear
interpolation Resizing layer within the model.

For imbalance handling, class_weight='balanced' was used instead of SMOTE.
This decision was driven by computational considerations: SMOTE would inflate
the training set from approximately 199,000 to 318,000 samples, and
MobileNetV2's 2D convolutions are substantially more expensive per sample
than the 1D CNN's operations. Class weighting achieves equivalent imbalance
correction without dataset inflation.

7.2 Analysis: Why the 1D CNN Outperforms MobileNetV2
------------------------------------------------------

The 1D CNN outperformed MobileNetV2 by a wide margin across every metric:
F1-score of 0.4333 versus 0.0291, ROC-AUC of 0.9486 versus 0.8758, and
PR-AUC of 0.7947 versus 0.3016. Four factors explain this result:

First, there is a fundamental domain mismatch between the source and target
tasks. MobileNetV2 was pre-trained on ImageNet, a dataset of natural images
containing objects such as dogs, cars, and buildings. The convolutional
filters it learned detect edges, textures, and spatial shapes — features
that are entirely irrelevant to tabular fraud data artificially arranged into
a 6×5 pixel grid. The "images" seen by MobileNetV2 bear no resemblance to
the natural images on which its feature extractors were optimized.

Second, the backbone was frozen and therefore could not adapt its
convolutional filters to the tabular domain. Only the small Dense
classification head was trainable, severely limiting the model's capacity to
learn task-specific representations.

Third, resizing the 6×5 input to 32×32 through bilinear interpolation
introduces artifacts and spatial correlations that do not exist in the
original data. This transformation adds noise while diluting the signal
contained in the original 30 features.

Fourth, the 1D CNN is purpose-built for this data representation. By treating
the 30 features as a one-dimensional sequence, Conv1D layers capture local
feature interactions without imposing artificial two-dimensional spatial
structure. This is a more natural and efficient representation for tabular
data than a 2D grid.

This comparison demonstrates that transfer learning is most effective when the
source and target domains share similar data characteristics. When the domains
are fundamentally different — as with natural images and tabular financial
data — a simpler, purpose-built architecture trained from scratch can
substantially outperform a larger pre-trained model.

8. ERROR & FAILURE ANALYSIS
================================================================================

A detailed error analysis was conducted on the default 1D CNN to understand
the nature and patterns of misclassification. The model produced 23 false
negatives (missed fraud) and 304 false positives (false alarms) on the test
set.

The most striking finding is that errors are overwhelmingly high-confidence.
False negatives have a mean predicted probability of fraud of only 0.0005,
with a median of 0.0. This means the model does not merely miss these
fraudulent transactions — it confidently labels them as legitimate with near-
complete certainty. Similarly, the most confident false positive has a
predicted probability of 1.0, indicating the model is fully convinced the
legitimate transaction is fraudulent. The model is not uncertain about its
errors; it is systematically deceived by edge cases that mimic the opposite
class.

Feature distribution analysis on key discriminative features (V14, V17, V12,
V10, V4, and Amount) reveals that false negative samples cluster within the
"normal" feature ranges typically associated with legitimate transactions.
These missed fraud cases lack the typical fraud signature — their PCA
component values fall within ranges that the model has learned to associate
with legitimate behavior. In essence, these are sophisticated fraudulent
transactions that, in feature space, are indistinguishable from legitimate
ones.

The false positives, with a mean confidence of 0.8641 and median of 0.9404,
represent legitimate transactions that happen to exhibit feature patterns
resembling fraud. Their mean normalized Amount of 0.03 suggests these are
typically low-value transactions, which may share characteristics with
certain fraud patterns.

These findings suggest that the model's error modes are not random but
reflect genuine ambiguity in the decision boundary: some fraud looks like
legitimate activity, and some legitimate activity looks like fraud. Addressing
these errors would likely require additional features beyond the PCA
components, such as behavioral or temporal patterns, or ensemble approaches
that combine multiple complementary classifiers.

11. CONCLUSION
================================================================================

This study investigated deep learning approaches for credit card fraud
detection on a highly imbalanced dataset containing 284,807 transactions with
only 0.172% fraud. A 1D CNN architecture was developed and systematically
evaluated alongside a Logistic Regression baseline and a MobileNetV2
foundation model.

The ablation study produced a notable finding: the best-performing model was
the 1D CNN trained without SMOTE oversampling, achieving an F1-score of
0.7930 with precision of 0.8248 and recall of 0.7635. This challenges the
common assumption that synthetic oversampling always benefits imbalanced
classification. The analysis suggests that SMOTE's synthetic samples can
introduce noise near the decision boundary, and that training on real
examples alone — when combined with appropriate architecture and
regularization — can yield superior precision without catastrophic recall
loss.

Batch Normalization was found to be essential for stable training, with its
removal causing F1 to drop to 0.2153. The foundation model comparison
demonstrated that MobileNetV2, despite its 0.4 million parameters and
ImageNet pre-training, achieved only F1=0.0291 — confirming that transfer
learning is ineffective when source and target domains are fundamentally
different. Error analysis revealed that model failures are high-confidence
rather than uncertain, with missed fraud transactions exhibiting feature
distributions indistinguishable from legitimate activity.

Several directions for future work are identified:

  1. Threshold tuning: Optimizing the classification threshold on a
     validation set rather than using the fixed 0.5 default could
     substantially improve the precision-recall trade-off.

  2. Feature engineering: Deriving hour-of-day from the Time feature and
     applying log-transformation to Amount could provide more discriminative
     input representations.

  3. Ensemble methods: Combining CNN predictions with Logistic Regression or
     gradient-boosted models (e.g., XGBoost) could leverage complementary
     strengths across different model families.

  4. Foundation model fine-tuning: Unfreezing select backbone layers in
     MobileNetV2 with a very low learning rate may allow partial adaptation
     to the tabular domain.

  5. Tabular-specific architectures: Models designed explicitly for tabular
     data, such as TabNet or FT-Transformer, may outperform both the 1D CNN
     and vision-based foundation models by natively handling heterogeneous
     feature relationships.
